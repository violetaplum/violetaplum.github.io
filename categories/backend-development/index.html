Event-Driven 뱅킹 서비스 구현하기: Go와 CQRS 패턴

소개 CQRS(Command Query Responsibility Segregation)와 Event Sourcing 패턴에 관심이 많았는데, 이번 토이 프로젝트를 통해 이 두 패턴을 함께 구현해 봤다. ‘초’ 간단한 뱅킹 서비스를 예시로 삼아, Go 언어로 개발하고 Kafka, PostgreSQL, OpenTelemetry와 같은 현대적인 기술 스택을 사용해 봤다. 프로젝트 아키텍처 이 프로젝트는 크게 다음과 같은 컴포넌트로 구성되어 있다: Account API: 계좌 생성, 입금, 출금 등의 명령(Command)을 처리하는 서비스 Event Processor: 이벤트를 소비하고 조회(Query) 모델을 업데이트하는 서비스 Postgres: 이벤트 저장소 및 조회 모델 데이터베이스 Kafka: 이벤트 메시징 시스템 (분산 처리) OpenTelemetry: 분산 추적 및 모니터링 시스템 Jaeger: 트레이싱 데이터 시각화 (UI 용도) 전체 아키텍처는 다음과 같은 흐름으로 동작한다:

  • Go
  • CQRS
  • Event Sourcing
  • Kafka
  • Microservices
  • OpenTelemetry
  • Docker
